## 1. 服务管理

### 查看 OpenClaw 状态

```bash
# systemd 状态（进程是否在跑）
systemctl --user status openclaw-gateway.service

# QQ Bot 频道状态
openclaw channels status

# 查看进程 PID
ps -ef | grep openclaw | grep -v grep
```

### 重启 OpenClaw

```bash
systemctl --user restart openclaw-gateway.service
```

> **注意**：`systemd failed` 状态不等于服务挂了。如果有手起实例（`nohup ... & disown`）占着端口，systemd 的 gateway 会自我保护退出（exit 78）——这是预期行为，不是错误。真正的服务在跑（`ps -ef` 看 PID 667471）。

### 手动启动（看门狗不会自动拉起）

```bash
cd /home/wwh/.openclaw
source ~/.openclaw/mcp-env/bin/activate
nohup python3 second_brain_mcp_server.py > /home/wwh/mcp_server.log 2>&1 & disown

# 验证
ps -ef | grep second_brain_mcp_server
```

> **注意**：kill 后 OpenClaw 不会自动拉起，必须手动执行上面的启动命令。

### 日志查看

```bash
# 最近 50 行
journalctl --user -u openclaw-gateway.service --no-pager -n 50 --since '5 min ago'

# 实时跟踪
journalctl --user -u openclaw-gateway.service -f

# MCP server 日志
tail -f /home/wwh/mcp_server.log
```

## 2. MCP Server

### 调用链路（必须记住）

```
QQ 消息 → OpenClaw qqbot → AI 识别"记：xxx"
  → 调 save_to_second_brain() MCP 工具
  → second_brain_mcp_server.py（stdio 子进程，懒加载）
  → git pull / add / commit / push
  → 响应用户（含 commit hash）
```

**关键**：QQ bot **不走** shell hook（`~/.openclaw/hooks/second-brain/*.sh`），走的是 MCP server。修改 MCP server 代码后**不需要重启 OpenClaw**——下次发"记：xxx"自动加载新代码。

### 定位实际进程

```bash
# 找 MCP server 实际运行的 Python 进程
ps -ef | grep mcp-venv | grep -v grep

# 找 MCP 配置 entry point
ls ~/.openclaw/config/*.json
cat ~/.openclaw/config/*.json | grep -A5 "second-brain"
```

### 确认 MCP 是否注册

```bash
# 检查配置文件
cat ~/.openclaw/openclaw.json | grep -A3 second-brain
```

应该看到：
```json
"second-brain": {
  "command": "/home/wwh/.openclaw/mcp-venv/bin/python",
  "args": ["/home/wwh/.openclaw/second_brain_mcp_server.py"]
}
```

## 4. 故障排查

### Q: 发消息后没反应

```bash
# 1. 看 gateway 状态
systemctl --user status openclaw-gateway.service

# 2. 看 QQ 频道
openclaw channels status

# 3. 看最近日志
journalctl --user -u openclaw-gateway.service --no-pager -n 30 --since '10 min ago'
```

### Q: MCP 工具调用失败

```bash
# 确认 MCP server 在跑
ps -ef | grep second_brain_mcp_server

# 确认已注册
cat ~/.openclaw/openclaw.json | grep second-brain

# 手动测试 MCP server
cd /home/wwh/.openclaw
source ~/.openclaw/mcp-env/bin/activate
python3 second_brain_mcp_server.py
# 看是否有报错
```

### Q: git push 失败

```bash
# 验证 SSH 认证
ssh -T git@github.com

# 看 git remote URL
git remote -v
# 应该看到：git@github.com:WWH3602/second-brain.git
```

### Q: 分类不准

见 `scripts/second_brain_mcp_server.py` 的 `classify()` 函数，优先级：
1. 人物（人名 + 社交动词）→ `02-人物/`
2. 软件工具（开源/软件/下载/安装等）→ `01-软件工具/`
3. 网络资源（URL 或爬虫关键词）→ `03-网络资源/`
4. 默认 → `00-Inbox/`
