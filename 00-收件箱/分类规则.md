# 第二大脑 · 分类规则手册（AI 必读）

> **用途**：当 QQ 收件箱有新笔记时，AI（耗子助手/小龙虾）按此规则自动分类入库
> **版本**：v1.0 | 2026-06-23
> **存放位置**：`00-收件箱/分类规则.md`（与收件箱同级，AI 每次整理前先读）

---

## 核心原则

1. **所有 QQ 消息先进 `00-收件箱/`**，不做实时分类
2. **定期（或收到「整理收件箱」指令时）** 按以下规则批量分拣
3. **每条笔记只属于一个分类**，选最匹配的
4. **无法确定时留在 `00-收件箱/`**，由用户人工判断
5. **省份/城市等地理层级信息**优先从笔记内容提取，提取不到则标 `待确认`

## 各目录详细规范

### 01-软件工具 — 按平台分

```
📁 01-软件工具/
├── README.md
├── PC软件/PC软件清单.md     ← Windows 电脑软件（用功能分类分段记录）
├── 安卓软件/安卓软件清单.md  ← Android 手机 App
└── 苹果软件/苹果软件清单.md  ← macOS / iOS 软件
```

**整理方式**：不创建独立文件，直接写入对应平台的 `XX软件清单.md`，在 `## 功能分类` 标题下新增条目。

**命中判断**：
| 内容特征 | → 平台 |
|:---|:---:|
| Windows/.exe/电脑/PC/图吧工具箱等 | PC软件 |
| Android/安卓/APK/手机应用 | 安卓软件 |
| macOS/Mac/苹果/iOS/Raycast等 | 苹果软件 |
| 跨平台/Web/在线工具 | PC软件（默认） |

**清单内格式**：
```markdown
### 软件名
- **用途**：一句话说明
- **网址**：https://...
- **来源**：QQ/耗子助手 YYYY-MM-DD
```

### 02-网页收藏

```
📁 02-网页收藏/
├── 网页收藏.md             ← 所有网站统一记录于此
└── edge收藏夹_*.html       ← 浏览器导出数据（保持原名）
```

**整理方式**：直接追加到 `网页收藏.md`，按主题分类分段。

**格式**：
```markdown
## 主题分类

- [网站名](URL) - 一句话说明（来源：YYYY-MM-DD）
```

### 03-美食探店 — 按省份分

```
📁 03-美食探店/
└── 📁 {省份名}/
    └── {城市}.md         ← 同城市所有美食整合到一个文件
```

**整理方式**：
1. 从笔记内容提取省份和城市名（如「福州达明路」→ 福建省/福州.md）
2. 如无城市信息 → `待确认/`
3. 追加到该城市的文件中，按 `## 店铺名` 分段

**内容格式**：
```markdown
# 福州

## 🥩 老牌依强牛肉店
- **地址**：详细地址
- **推荐菜**：必点菜品
- **来源**：QQ/耗子助手 YYYY-MM-DD
```

## 特殊值约定

| 场景 | 处理方式 |
|:---|:---|
| 提取不到省份/城市 | `待确认/` 子目录 |
| 软件无法确定平台 | 默认 PC软件 |
| 同时命中多个规则 | 按优先级 P0→P9 取最高 |
| 内容为空 / 仅测试 | `05-技术备忘/链路测试/` |

## 错误案例分析

| 之前的问题 | 原因 | 现在怎么避免 |
|:---|:---|:---|
| 张三聚餐笔记存了 2 份 | 重复保存，无去重逻辑 | 入库前检查目标目录是否已有同名文件 |
| note.md 无意义文件名 | 未重命名 | 按格式 `YYYYMMDD-主题.md` 重命名 |
| 同一条笔记在 Inbox 和归档同时出现 | 分类后未删除原文 | 移动（不复制）文件 |
